Dynamic Authentication by 'Risk Level Analysis Based User Profiling' Using Machine Learning :



Parth Khaladkar
May 7 2019



This was the project I created back in the final year of my undergrad studies.This was a group project.The system aims to analyse and learn from user behaviour patterns to reduce the probabiliy of unwarranted access.

A user login record is analyzed using eight user parameters(Browser, OS, Time, IP, Device, FailedAttempts, Location, Timezone) which are collected during current login attempt.Each parameter has a respective weight attached to it based on it's importance.

A user is profiled according to his historical patterns and classified as genuine or fradulent before logging in using login credentials and attributes. These attributes are analyzed using Machine Learning model consisting of three trained modules through which risk level is calculated and used to determine the authentication techniques in different combinations for a particular user.

So if a current record has parameters which do not resemble the profile the risk score will get incremented in accordance to respective weight of the parameter. Use of three algorithms increases accuracy of classification. So, a fradulent user will have to go through multiple generated set of authentication techniques (like OTPs, Email Verification, Graphical password, pattern match)while a genuine one will not.

System Diagram



The system consists of three main modules

  • Data Generation Module :
    This includes user account creation(Take in initial user entered credentials), parameter extraction, data preprocessing, fetchingg user history and updating the dataset with current login parameters.

  • Risk Engine :
    This contains the main buisness logic.It works on the user dataset and computes the probability and risk score. It has three machine learning algorithms
      1.SVM (Support Vector Machine)
      2.One Class SVM
      3.Naive-Bayes classifier

  • Authentication Module :
    The authentication module with the help of the risk score by one-class SVM and probabilities of being fraud by SVM and Naive-Bayes' classifier computes the risk level, randomly presents the user with the authentication methods. The user thus gets classified according to the risk level he is assigned into. One to Four signifies risk threat or probability of the user being fradulent from minimum to maximum.
